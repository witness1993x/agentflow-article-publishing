# Example: Browser Agent For Everyone

## 1. Conclusion

`Decision`: `drop`

`One-line thesis`: “面向所有人的通用浏览器代理”叙事很大，但当前没有足够清晰、可控、可获客的切入口，对小团队来说机会成本过高。

`Confidence`: `medium`

`Timing verdict`: `late`

`Primary constraint`: `team_side`

## 2. Opportunity

- Opportunity ID: OPP-003
- Opportunity: 面向大众用户的通用浏览器操作代理
- Original question: 要不要做一个人人都能用的 browser agent？
- Reframed question: 小团队是否有机会在通用浏览器代理这个方向上找到一个未被巨头与平台能力吞没的切入口？
- Why now: 浏览器自动化、模型能力和 agent 叙事都在升温
- Why us: 团队懂 agent workflow，但不拥有浏览器分发和平台控制力
- Falsified if: 存在一个明显的垂直场景，用户愿意为了稳定自动化付费

## 3. Gate Summary

| Gate | Verdict | Score | Why it passed / held / failed |
|---|---:|---:|---|
| Reframe | pass | n/a | 已明确问题不是“能不能做”，而是“有没有可赢切口” |
| Signal | pass | 4 / 5 | 市场对 browser agent 的关注度很高 |
| Demand Window | pass | 3 / 5 | 确实有需求，但需求过于泛化，不指向具体切口 |
| Product Window | hold | 2 / 5 | 巨头、平台和基础模型层都可能快速吞掉通用能力 |
| Action | fail | 1 / 5 | 当前团队缺少分发、资源和结构性优势 |

## 4. Strongest Evidence

1. 用户和开发者都对 browser agent 方向有明显关注。
2. 技术能力在快速进步，演示效果越来越强。
3. 这个方向天然有想象力，容易形成外部关注。

## 5. Biggest Risks

1. 通用入口会被浏览器、操作系统或大模型原生能力吸收。
2. 小团队在分发和稳定性上没有防守能力。
3. 演示价值很强，但实际产品长期可用性与商业化较弱。

## 6. Kill Signals

- 如果始终无法收敛到一个高价值垂直场景。
- 如果获客必须依赖高投入教育市场。
- 如果核心能力被平台或大模型原生集成快速覆盖。

`Veto from gate`: `Action`

## 7. Decision Logic

### Why this may work

- 如果能找到垂直高价值流程，理论上仍有产品机会。
- 市场教育已经在被更大玩家推进。

### Why this may fail

- “面向所有人”本身就是一个过宽的错误问题。
- 当前团队没有足够强的非对称优势支撑下注。

### Why now / why not now

- 现在适合关注这个大方向。
- 现在不适合由当前团队直接做一个泛化产品。
