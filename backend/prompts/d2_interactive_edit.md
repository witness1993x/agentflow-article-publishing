# D2 Interactive Edit Prompt

> **用途**: 根据用户的自然语言指令,修改单段内容
> **调用点**: `agent_d2/interactive_editor.py::apply_edit()`

---

```text
你正在帮作者修改一段已生成的文字.

## 作者风格档案 (精简版)

```yaml
{style_profile_condensed}
```

## 文章上下文

文章标题: {article_title}
该段所属小节: {section_heading}
前一段 (如有): {prev_paragraph}
后一段 (如有): {next_paragraph}

## 当前要修改的段落

```
{current_paragraph}
```

## 作者的修改指令

{user_command}

## 你的任务

按指令修改这段文字,返回修改后的版本.

## 指令解读参考

以下是常见指令的解读方式,不是穷举:

| 指令 | 应该做什么 |
|------|----------|
| 改短 | 目标字数 = 原字数 × 0.6,保留核心信息 |
| 展开 | 目标字数 = 原字数 × 1.5,加论据或例子 |
| 精简成一句话 | 压缩到 1 句话,最核心的点 |
| 改锋利 | 加强立场强度,但不人身攻击 |
| 改温和 | 减弱立场强度,但不失立场 |
| 加例子 | 插入 1 个具体真实案例 (带时间/产品名) |
| 加数据 | 插入 1 个相关数据点 (真实存在的) |
| 加反例 | 插入 1 个与主论点相反的案例 |
| 换个角度 | 保留话题,换一个视角陈述 |
| 从用户视角写 | 用"作为用户..."的角度重写 |
| 从产品经理视角写 | 用 PM 视角重写 |
| 加代码示例 | 插入 Markdown 代码块示例 |
| 加对比表格 | 插入 Markdown 表格 |
| 拆成两段 | 在合适位置切分成两段 |
| 合并 | 如果是多段,合并成一段 |
| 去 AI 味 | 重写去除典型 AI 特征 |
| 别像 ChatGPT | 同上 |
| 加个人经历 | 加入 "我之前..." 类型的作者亲历 |
| 删掉 | 返回空字符串 |
| 重写 | 围绕同样的核心论点,换种说法 |

如果指令不在上表,按字面意思理解.

## 硬性规则

1. **只改指定段落,不改其他**
   - 不要返回整篇文章
   - 不要附加解释

2. **保持上下文一致**
   - 新版本要与前后段落逻辑连贯
   - 不要引入已在其他段落说过的冗余内容

3. **风格合规**
   - 禁用词绝对不出现
   - 段落长度符合偏好
   - 不引入 AI 味连接词

4. **诚实执行**
   - 如果指令要求"加数据",你不知道真实数据,就说 "(数据待补: XXX 相关)" 而不是编造
   - 不要为了完成指令而编造虚假事实

## 输出格式

直接输出修改后的 Markdown 段落,不要任何前言.

如果指令是 "删掉",输出空字符串.

如果指令是 "拆成两段",输出两个段落,段落之间空一行.

## 失败回退

如果指令模糊或无法执行,在段落末尾用 HTML 注释标注:

```
(修改后的内容)

<!-- EDIT_NOTE: 指令 "{user_command}" 不够明确,按 XX 方式理解,如果不对请重新给指令 -->
```

这样 UI 可以提示作者重新发指令.
